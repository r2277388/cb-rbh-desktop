Retrieve Rolling Report Data Sources

This procedure collects Sam's data-source retrieval notes for recurring rolling reports. Use it when a report needs a fresh weekly source file before the launcher can update a cache or build the final workbook.

The screenshots are reference images from the original PDF directions. Follow the written steps below for the working process, source folders, and launcher actions.

Existing reportsReaderlink, Target NOC, Barnes & Noble.
Planned reportsAWBC, Edelweiss, XGap. These source steps are captured before the rolling reports are built.
General ruleRetrieve the source file first, save it in the matching DataWarehouse weekly folder, then run the report's cache/update process.

Readerlink Existing

Report: Readerlink Rolling Reports

Source PDF: F:\ANALYSIS\Finance\DataWarehouse\Atelier Readerlink\Readerlink_DataSources.pdf

Working source folders:

  • Sales workbooks: F:\ANALYSIS\Finance\DataWarehouse\Weekly reports\2026\Readerlink
  • Inventory workbooks: F:\ANALYSIS\Finance\DataWarehouse\Weekly reports\2026\Readerlink\Inventory
  • Cache folder: F:\ANALYSIS\Finance\DataWarehouse\Atelier Readerlink\cache

Steps

  1. Retrieve the latest Readerlink sales export for the new reporting week.
  2. Before saving/exporting, verify the selected week is correct and no extra filters are applied.
  3. Save the sales workbook in the current-year Readerlink weekly source folder.
  4. Retrieve the matching Readerlink inventory export and save it in the Readerlink Inventory folder.
  5. Run Retailer Rolling Reports > Readerlink Rolling Reports > Add a new week's data to cache.
  6. Review the selected source files shown by the cache update prompt before confirming.
  7. Build the Readerlink Rolling Report after the cache update completes.

Barnes & Noble Existing

Report: Barnes & Noble Rolling Reports

Source PDF: F:\ANALYSIS\Finance\DataWarehouse\Atelier BarnesNoble\BN DataSources.pdf

Default raw source base: F:\ANALYSIS\Finance\DataWarehouse\Weekly reports\2026\Barnes & Noble

Steps

  1. Retrieve the new weekly Barnes & Noble raw source files from the B&N portal/source shown in the screenshots.
  2. Create or use the weekly raw folder for the new week under the Barnes & Noble weekly reports folder.
  3. Save the sales, inventory, and non-book source files into that weekly raw folder using the current naming convention.
  4. Run Retailer Rolling Reports > Barnes & Noble Rolling Reports.
  5. At the folder review screen, confirm the launcher found the expected raw folder and source workbooks. Use Change raw folder if it selected the wrong week.
  6. Run the B&N process steps in order: refresh/combine raw source data, build working sales/inventory files, then run the final rolling report.
